# Инструкция по установке и запуску

## Шаг 1: Установка зависимостей

Установите Python 3.8 или новее, затем выполните:

```bash
pip install -r requirements.txt
```

## Шаг 2: Получение API credentials

1. Перейдите на https://my.telegram.org
2. Войдите в свой аккаунт
3. Перейдите в раздел "API development tools"
4. Создайте новое приложение (если еще не создавали)
5. Скопируйте:
   - `api_id` (число)
   - `api_hash` (строка)

## Шаг 3: Настройка окружения

Создайте файл `.env` в корне проекта со следующим содержимым:

```
API_ID=ваш_api_id
API_HASH=ваш_api_hash
SECRET_KEY=любая_случайная_строка_для_безопасности
DEBUG=True
FLASK_PORT=5000
FLASK_HOST=0.0.0.0
```

**Важно**: Замените `ваш_api_id` и `ваш_api_hash` на реальные значения.

## Шаг 4: Запуск приложения

```bash
python app.py
```

Приложение будет доступно по адресу: **http://localhost:5000**

## Шаг 5: Использование

1. Откройте http://localhost:5000 в браузере
2. Отсканируйте QR-код через приложение Telegram:
   - На телефоне откройте Telegram
   - Перейдите в Настройки → Устройства
   - Нажмите "Привязать устройство" или "Сканировать QR-код"
3. После сканирования на странице отобразится ваш профиль
4. Бот автоматически активируется и будет отвечать на все сообщения эхом
5. Для выхода нажмите кнопку "Выйти"

## Устранение проблем

**Ошибка "API_ID is not set"**
- Проверьте файл `.env` и убедитесь, что API_ID и API_HASH заполнены

**QR-код не генерируется**
- Убедитесь, что у вас есть активное интернет-соединение
- Проверьте правильность API credentials

**Бот не отвечает на сообщения**
- Убедитесь, что бот был успешно активирован (должна появиться отметка "✓ Бот активирован")
- Проверьте логи в консоли на наличие ошибок

## Структура проекта

- `app.py` - основной Flask сервер
- `auth_manager.py` - управление авторизацией через QR-код
- `userbot_manager.py` - управление юзерботом
- `config.py` - конфигурация приложения
- `templates/index.html` - веб-интерфейс
- `static/style.css` - стили
- `static/main.js` - JavaScript логика
- `sessions/` - файлы сессий Telegram (создается автоматически)

## Безопасность

⚠️ **Важно**: 
- Не делитесь файлом `.env` с другими
- В продакшене установите `DEBUG=False`
- Используйте сильный `SECRET_KEY`
- Файлы сессий содержат конфиденциальные данные
